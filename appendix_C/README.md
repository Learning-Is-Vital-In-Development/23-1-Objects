# C. 동적인 협력, 정직인 코드

객체는 동적이다.

프로그램은 정적이다.

프로그래머가 객체지향 프로그램을 작성하기 위해서는 두 가지 모델을 동시에 그려야함

- 프로그램 실행 구조를 표현하는 움직이는 모델
- 코드의 구조를 담는 고정된 모델

- 동적 모델 → 객체 + 협력
- 정적 모델 → 타입 + 관계

우선되어야 되는 점은

동적 모델이 더 중요함.

정적 모델은 동적 모델에 의해 주도돼야 하고

동적 모델이라는 토대 위에 새워져야 함

→ 결국 수정이 용이하게 하려면 동적 모델을 기반으로 해야된다.
